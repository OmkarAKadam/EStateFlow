# Project Goal and Problem Definition

## 1. Problem Statement

In city-level rental markets (e.g., Vadodara), students and working professionals often spend 2–5 days physically searching for rental properties. This typically involves visiting brokers, traveling across multiple areas, and paying brokerage fees.

### Tenant Challenges

- Limited visibility of available properties  
- Time-consuming physical visits  
- Dependence on local brokers  
- Brokerage charges equal to 1–2 months’ rent  

### Small Landlord Challenges

- Reliance on brokers to find tenants  
- Commission losses  
- Lack of a simple digital platform for independent listing  

### Core Problem

There is no centralized, broker-free, city-focused digital platform that allows landlords to list properties and tenants to search and contact them directly.

---

## 2. Target Users

### Primary Users

- College students relocating for education  
- Working professionals relocating for employment  
- Budget rental seekers (₹5,000–₹20,000 per month range)  

### Secondary Users

- Individual property owners  
- Small landlords renting flats, rooms, or houses  
- Independent property sellers  

This platform is not intended for large real estate companies or commercial builders.

---

## 3. Value Proposition

### Traditional Process

- Visiting multiple brokers  
- Traveling across different areas  
- Paying 1–2 months’ brokerage  
- Spending multiple days searching  

### Platform-Based Process

- View all listed properties in one place  
- Filter by price, area, and property type  
- Preview property images before visiting  
- Contact owners directly  
- Reduce search time and brokerage expenses  

### Quantifiable Impact

- Saves approximately 2–5 days of physical search  
- Reduces brokerage costs (₹10,000–₹30,000 depending on rent)

---

## 4. Project Purpose

### Immediate Objective

- Internship submission  
- Academic evaluation  
- Portfolio-level full stack system  

### Potential Future Direction (Optional)

- Expansion into a production-ready city-based rental platform  
- Extension into a mobile application  

The current goal is to build a technically sound, deployable, and well-architected full stack system.

---

## 5. Definition of Success

The project is considered successful when it includes:

- User registration and login  
- Role-based access control (Owner / Tenant)  
- Property CRUD functionality  
- Image upload capability  
- Property filtering  
- Owner contact information display  
- Clean and responsive user interface  
- Fully deployed live application  
- Public GitHub repository with structured documentation  

### Minimum Acceptable Outcome

A fully deployed and functional demo that clearly demonstrates complete full stack architecture from frontend to database.
